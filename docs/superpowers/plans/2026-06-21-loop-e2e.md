# loop-e2e CLI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 稼働中Webアプリに対し、AI生成シナリオでサイト構造化・差分検出・多面検証・レポート/Issue起票・フィードバック反映を行うTypeScript製CLIを構築する。

**Architecture:** `commander` の4サブコマンド（init/scenario/run/feedback）が `pipeline`（collect→diff→verify→report）を駆動し、`services`（github/db/browser/llm/state）を利用する。LLM(Claude)出力は zod 検証して取り込む。状態は `.loop-e2e/` に永続化し差分基準(baseline)とする。

**Tech Stack:** TypeScript, Node 20+, pnpm, commander, @clack/prompts, zod, @anthropic-ai/sdk, playwright, pg, mysql2, @octokit/rest, yaml, dotenv, pino, vitest.

## Global Constraints

- Node.js >= 20、TypeScript strict、ESM、pnpm。
- 機密（ANTHROPIC_API_KEY / GITHUB_TOKEN / DBパスワード / ログイン情報）は `.env` のみ。config・ログ・レポート・Issue へ機密を出力しない（マスキング必須）。
- 全LLM出力は zod スキーマで検証してから利用。検証失敗時はリトライ（指数バックオフ、最大3回）。
- 不変データ方針: オブジェクトは複製で更新、破壊的変更をしない。
- 1ファイル200–400行目安・最大800行。1ファイル1責務。
- テストカバレッジ 80%+。LLM/外部I/Oはモック。
- DB対応は postgres と mysql の両方必須。アダプタで抽象化。
- 設定ファイルは `loop-e2e.config.yaml`、シナリオは `<scenarioDir>/*.scenario.yaml`。
- **モデル使い分け**: 計画・分析・判断＝Opus、レポート生成＝Sonnet、反証検証＝Opus。`config.models={planning,report,verification}` で上書き可。LLMクライアントは役割→モデルIDを解決して呼ぶ。
- **baseline**: 既定 非コミット。`config.baseline.commit=true` のときのみ `.loop-e2e/baseline/` を git 管理対象にする（`.gitignore` を条件分岐）。
- **Issue自動起票ゲート**: 反証検証(Opus)で `classification∈{bug,unnecessary}` かつ反証を耐えたもののみ起票。`uncertain` はレポートのみ。
- **シナリオ要件入力**: 対象リポジトリの README/docs/ソースコード/Git ログを自動収集（＋任意 `--from`）。ソースは要約/関連抽出でトークン制御。

参照スペック: `docs/superpowers/specs/2026-06-21-loop-e2e-design.md`

## マイルストーン概要（各々が動作・テスト可能な増分）

1. **M1 基盤** — 雛形・lint/test・config スキーマ/ロード/保存・util・ログ
2. **M2 init** — 対話設定→config/.env生成・scenarioDir作成・GitHubラベル作成・状態ディレクトリ
3. **M3 collect** — Playwright巡回＋ページ構造化・state store・baseline保存
4. **M4 scenario** — Claudeでシナリオ＋想定結果＋想定DB状態生成
5. **M5 diff + report** — 差分検出・レポートmd/json・GitHub Issue起票
6. **M6 verify** — layout/security/conditional/registered-data/error-handling 5検証
7. **M7 feedback** — 取込・検証・反映（baseline/シナリオへ）＋統合/E2E仕上げ

> M1・M2 は完全なbite-sizedTDD手順で記述。M3–M7 は「ファイル・インターフェース・成果物・書くべきテスト」を確定した**タスク定義**として記述し、実行直前（基盤確定後）に同形式のbite-sizedステップへ展開する。各タスクは独立してテスト可能な成果物で終わる。

```typescript
export function maskSecrets(text: string, secrets: string[]): string {
  return secrets.filter(Boolean).reduce(
    (acc, secret) => acc.split(secret).join('***'),
    text,
  )
}
```

```typescript
export async function ensureDir(path: string): Promise<void> { await mkdir(path, { recursive: true }) }
```

```typescript
// schema.ts (抜粋)
import { z } from 'zod'
export const RepositorySchema = z.object({
  name: z.string(), label: z.string(), url: z.string().url(),
  role: z.enum(['frontend', 'backend']), audience: z.enum(['user', 'admin']),
})
export const AuthSchema = z.object({
  strategy: z.enum(['form', 'basic', 'none']),
  loginPath: z.string().optional(), usernameEnv: z.string().optional(), passwordEnv: z.string().optional(),
})
export const TargetSchema = z.object({ name: z.string(), baseUrl: z.string().url(), auth: AuthSchema.optional() })
export const DbSchema = z.object({
  name: z.string(), type: z.enum(['postgres', 'mysql']),
  host: z.string(), port: z.number().int().positive(), database: z.string(), user: z.string(), passwordEnv: z.string(),
})
const ModelsSchema = z.object({
  planning: z.string().default('claude-opus-4-8'),
  report: z.string().default('claude-sonnet-4-6'),
  verification: z.string().default('claude-opus-4-8'),
}).default({})
export const ConfigSchema = z.object({
  repositories: z.array(RepositorySchema).min(1),
  targets: z.array(TargetSchema).min(1),
  databases: z.array(DbSchema),
  schedule: z.object({ intervalMinutes: z.number().int().min(1) }),
  scenarioDir: z.string().min(1),
  github: z.object({ labels: z.object({ ready: z.string(), autoDetect: z.string() }) }),
  baseline: z.object({ commit: z.boolean().default(false) }).default({}),
  models: ModelsSchema,
})
export type Config = z.infer<typeof ConfigSchema>
export const CONFIG_FILENAME = 'loop-e2e.config.yaml'
```
